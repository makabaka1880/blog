::DefBox{id=Intersection}
If $A$ and $B$ are sets, then the intersection of $A$ and $B$, denoted $A \cap B$, is the set of elements that the two sets have in common. In other words, for an object to be an element of $A \cap B$, it must be an element of both $A$ and $B$.
$$
A, B : \{U\} \triangleright A \cap B := \lambda x : U. x \in A \wedge x \in B : \{U\}
$$
```lean
def intersection { U } (A B : Set U ) : Set U :=
    fun x => x ∈ A ∧ x ∈ B
```
The set theory game provides the definition in a double implication:
:::TheoremBox{id="Definition of Intersection via Membership"}
```lean
theorem mem_inter_iff {α} (x : α) (a b : Set α) 
    : x ∈ a ∩ b ↔ x ∈ a ∧ x ∈ b 
        := by rfl
```
:::
::

## 0x00. And
::QaBox
Suppose $x \in A$ and $x \in B$. Then $x \in A$.
```lean
example (x : U) (A B : Set U) (h : x ∈ A ∧ x ∈ B) 
    : x ∈ A := by sorry
```
::

Finally, we begin to step into the realm of logical connectives! The first connective we see is the logical conjunction : **And**. But before we dive into what we can do with conjunctions, we first need to understand what conjunctions are.

Conjunctions express the coexistence of two truths. A term of type $P \wedge Q$ provides both the proof of $P$ and the proof of $Q$, where $P$ and $Q$ are called **conjuncts**. In a sense, proof of conjunctions are just a pair of proofs for the left and right conjuncts. In Lean, given the proof $h_1 : P$ and $h_2 : Q$, you can construct the proof $\langle h_1, h_2 \rangle : P \wedge Q$ by using angle brackets.

::HintBox
You can type $\langle$ and $\rangle$ by typing a backslash, `<` or `>`, then moving the cursor away or typing anything.
:Pic{src="Screen Recording 2026-04-22 at 08.04.36.gif" alt="This also works in VSCode once you install the Lean 4 extension"}
::

::NoteBox
Lean uses church's second order encoding of logic. Therefore, the conjunctive construct $\langle \cdot, \cdot \rangle$ is formally

$$

a : P, b : Q \triangleright \langle a, b \rangle := \lambda C : \ast. \lambda f : P \to Q \to C. f\ a\ b : P \wedge Q

$$
::

::ExampleBox
The formal proof

::FlagDeriv{center=true}

#flag
$\ast : \square$

#pole
:::FlagDeriv

#flag
$h_P : P$

#pole

::::FlagDeriv

#flag
$h_Q : Q$

#pole
$\langle h_P, h_Q \rangle : P \wedge Q$


:::
::
Is idiomatically formalized in lean as:
```lean
example { P Q } (hP : P) (hQ : Q)
    : P ∧ Q := 〈hP, hQ〉
```
We'll see later tactics for reducing a conjunctive goal, like `constructor`. In fact, `constructor` was already provided in the previous chapters, but we never had the opportunity to use it.
::

Some of you familiar with software engineering and category theory might immediatly point out that since we're modeling the proof of conjunction as a tuple / product, proving any of the conjuncts is just a projection.

> For any of you who dont know what a projection is, it is a map that extracts an component from a tuple or pair.
> $$ 
>    \pi_1 : (x_1, x_2) \mapsto x_1 \\
>    \pi_2 : (x_1, x_2) \mapsto x_2 \\
> $$

In lean, projections are called using the methods `left` and `right`.

```lean
def And.left : A ∧ B → A := sorry
def And.right : A ∧ B → B := sorry
```

> In reality, `left` and `right` are not defined manually. They are generated by Lean automatically from the desugarization of `And`'s inductive declaration. We wont get into inductive type theory for now.

In Lean there is this very nice syntatic sugar that allows you to write method calls using the familiar dot operator from OOP languages. For example, `left h` can be written as `h.left`.

So the proof for this level is trivial:

::QaBox{type=answer}
```lean
example (x : U) (A B : Set U) (h : x ∈ A ∧ x ∈ B) 
    : x ∈ A := by exact h.left
```
:Qed

:::WarningBox
Again, since STG4 overrides many of Mathlib's original configurations, `exact left h` **does not work**. The namespacing of `And.left` is not opened for use in the game for encapsulation purposes.
:::
::

## 0x01. Element of an Intersection
::QaBox
Assume $x \in A \cap B$. Then $x \in B$.
```lean
example (x : U) (A B : Set U) (h : x ∈ A ∩ B) 
    : x ∈ B := by sorry
```
::

We see an intersection in $h$. Because we have no other way of extracting that information, let's try rewriting the definition of intersection at $h$.

$$
\left.\begin{align*}
\text{Objects} \quad 
A, B &: \{U\} \\
x &: U \\
\text{Assumptions} \quad
h &: (A \cap B)\ x
\end{align*}\right\}
\vdash B\ x
$$

After doing `rw [mem_inter_iff] at h`, we obtain

$$
\left.\begin{align*}
\text{Objects} \quad 
A, B &: \{U\} \\
x &: U \\
\text{Assumptions} \quad
h &: A\ x \wedge B\ x
\end{align*}\right\}
\vdash B\ x
$$

And the rest is just closing the goal on a right projection of $h$.

::QaBox{type=answer}
```lean
example (x : U) (A B : Set U) (h : x ∈ A ∩ B) 
    : x ∈ B := by 
        rw [mem_inter_iff] at h
        exact h.right
```
::

## 0x02. Intersection is a subset
::QaBox
For any sets $A$ and $B$, $A \cap B \subseteq A$.
```lean
example (A B : Set U) : A ∩ B ⊆ A := by sorry
```
::

This one is fairly easy. Just `intro x hx` to unwrap the subset clause and rewrite the definition of intersection at `hx`. Then left project to close the goal.

::QaBox{type=answer}
```lean
example (A B : Set U) : A ∩ B ⊆ A := by
    intro x hx
    rw [mem_inter_iff] at hx
    exact hx.left
```
:::NoteBox
Actually the `rw` step is completely optional. Lean is smart enough to deduce `hx.left` as a proof of $x \in A \cap B$.
:::
::

## 0x03. Proving a Conjunction
::QaBox
Suppose $x \in A$ and $x \in B$. Then $x \in A \cap B$.
```lean
example (x : U) (A B : Set U) (h1 : x ∈ A) (h2 : x ∈ B) 
    : x ∈ A ∩ B := by sorry
```
::

As we've said, Lean's unification engine (the "smart" part of the elaborator) is smart enough to see $x \in A \cap B$ as equivalent to $x \in A \wedge x \in B$. Therefore we just need to prove $x \in A \and x \in B$.

The $\langle \cdot, \cdot \rangle$ notations has been introduced in the previous level. However, the formal name for this construct is call **And Introduction**. We can call it using `And.intro`.
::QaBox
```lean
example (x : U) (A B : Set U) (h1 : x ∈ A) (h2 : x ∈ B) 
    : x ∈ A ∩ B := by exact And.intro h1 h2
```
Or using brackets
```lean
example (x : U) (A B : Set U) (h1 : x ∈ A) (h2 : x ∈ B) 
    : x ∈ A ∩ B := by exact ⟨h1, h2⟩
```
::

## 0x04. Subset of an Intersection
::QaBox
Suppose $A \subseteq B$ and $B \subseteq C$. Then $A \subseteq B \cap C$.
```lean
example (A B C : Set U) (h1 : A ⊆ B) (h2 : A ⊆ C) 
    : A ⊆ B ∩ C := by
```
::

Let's first unwrap the goal by `intro x hAx`.
$$
\left.\begin{align*}
\text{Objects} \quad
A, B, C &: \{U\} \\
x &: U \\
\text{Assumptions} \quad
h_1 &: A \subseteq B \\
h_2 &: A \subseteq C \\
h_{Ax} &: A\ x
\end{align*}\right\}
\vdash (B \cap C)\ x
$$

Let's introduce another tactic: `constructor`. The constructor does structural induction on inductive datatypes, and for a conjunctive goal, it just creates a subgoal for each conjunct.

$$
\left.\begin{align*}
\text{Objects} \quad
A, B, C &: \{U\} \\
x &: U \\
\text{Assumptions} \quad
h_1 &: A \subseteq B \\
h_2 &: A \subseteq C \\
h_{Ax} &: A\ x
\end{align*}\right\}
\vdash B\ x \quad (\text{Case left})
$$
$$
\left.\begin{align*}
\text{Objects} \quad
A, B, C &: \{U\} \\
x &: U \\
\text{Assumptions} \quad
h_1 &: A \subseteq B \\
h_2 &: A \subseteq C \\
h_{Ax} &: A\ x
\end{align*}\right\}
\vdash C\ x \quad (\text{Case right})
$$

The rest is simple. For each case, just apply $h_{Ax}$ to either $h_1$ or $h_2$.

::QaBox{type=answer}
```lean
example (A B C : Set U) (h1 : A ⊆ B) (h2 : A ⊆ C) 
    : A ⊆ B ∩ C := by
    intro x hAx
    constructor

    exact h1 hAx

    exact h2 hAx
```
:Qed
::

## 0x05. Intersection Subset of Swap
::QaBox
Prove the following theorem

:::TheoremBox{id="Intersection of Subsets"}
For any sets $A$ and $B$, $A \cap B \subseteq B \cap A$.
:::

```lean
theorem inter_subset_swap (A B : Set U) 
    : A ∩ B ⊆ B ∩ A := by sorry
```
::

Still pretty simple. Just expands the subset relation, use constructor to break down the membership of intersections, and projection

::QaBox{type=answer}
```lean
theorem inter_subset_swap (A B : Set U) 
    : A ∩ B ⊆ B ∩ A := by
        intro x hABx
        constructor
        exact hABx.right
        exact hABx.left
```
:Qed
::

## 0x06. Intersection is Commutative
::QaBox
Prove the following theorem

:::TheoremBox{id="Intersection is Commutative"}
For any sets $A$ and $B$, 
:::

```lean
theorem inter_subset_swap (A B : Set U) 
    : A ∩ B = B ∩ A := by sorry
```
::

By applying the subset antisymm property to the goal we arrive at two subgoals that could be each be proved using `inter_subset_swap`.

::QaBox{type=answers}
```lean
theorem inter_subset_swap (A B : Set U) 
    : A ∩ B = B ∩ A := by
        apply Subset.antisymm
        apply inter_subset_swap
        apply inter_subset_swap

```
